This page describes how the Ultrahaptics arrays work and gives an overview of how to use the SDK.

Mid-Air Haptics

The Ultrahaptics arrays are phased arrays of ultrasonic speakers. They transmit ultrasonic waves that are timed to coincide at a given point in space.

By sending a continuous wave the pressure at the focal point can be made to oscillate at 40 kHz. However this frequency is too fast to feel. In order to create a haptic point that you can feel, this 40 kHz signal is modulated with a sine wave at 200 Hz, so that the pressure at the focal point (also known as a control point in this SDK) varies as shown in this graph.

Using a hand tracker (Ultrahaptics development kits come with a Leap Motion Controller) we can position the focal point on your palm. More complex shapes like lines and circles can be created by rapidly scanning the control point or by using multiple control points simultaneously.

The Solver

When multiple control points are used, their fields can destructively interfere with each other and reduce their strength. Since the phase of the 40 kHz signal is unimportant to the haptic sensation, the Ultrahaptics SDK includes a solver that optimises the phases of each control point so that they constructively interfere as much as possible and give the strongest sensations possible.

Amplitude Modulation vs Spatial Modulation

Modulating a stationary control point with a sine wave envelope is not the only way to create a haptic sensation. Another way is to keep the control point unmodulated (i.e. its modulation amplitude is always 1) and to rapidly scan it along a path. Because the control point is quickly moved over each point on the path, the pressure at each point is effectively modulated and can be felt.

This spatial modulation is more efficient than amplitude modulation but is only applicable for paths (for points you will need to use amplitude modulation) and is not supported by all Ultrahaptics hardware.

APIs

When sending data to the array in order for it to emit ultrasound there are two APIs to choose from. Ultrahaptics::AmplitudeModulation::Emitter which is the simplest and always uses sinusoidal amplitude modulation, and Ultrahaptics::TimePointStreaming::Emitter which allows arbitrary modulation signals, allowing the use of amplitude modulation and spatial modulation.

Amplitude Modulation API

The Amplitude Modulation API only supports amplitude modulation. With this API you send updates at any time that contain a list of control points, including their position, modulation amplitude and frequency. These updates will be applied immediately and will persist until a further update is issued or the emitter is stopped. With the Amplitude Modulation API a sine modulation is always used (as shown in the graphs above).

// Create two control points at the given positions, amplitudes and frequencies
update([ {x=0.03m, y=0.0m, z=0.1m, amplitude=1.0, frequency=200Hz}, {x=-0.03m, y=0.0m, z=0.1m, amplitude=0.5, frequency=200Hz} ])
// These will continue to be emitted...
wait(1 second)
// ...until we say otherwise.
stop()

Shapes can be made by moving the control points in a loop.

loop:
for x in [0.00, 0.02, 0.04, 0.06]:
update([{x=x, y=0.0m, z=0.1m, amplitude=1.0, frequency=200Hz}])
wait(10 milliseconds)

If more than one control point is used they will automatically be split into two control groups out of phase by 180° as described above.

Time Point Streaming API

The Time Point Streaming API is more powerful but harder to use. It can be used for amplitude modulation or spatial modulation. It works in a similar way to multichannel USB audio. You provide a callback function that is called in a loop. In the callback you provide the position and instantaneous modulation amplitude of each control point. This allows you to use any modulation envelope - not just a sine wave as with Amplitude Modulation.

The API is called Time Point Streaming because it allows you to know the exact point in time that each sample will be emitted.

The TimePointStreaming_Circle example shows how to do spatial modulation of a circle, and the TimePointStreaming_Focus example shows how to do amplitude modulation of a point.

On UHEV1, the Amplitude Modulation SPI is actually implemented on top of Time Point Streaming, which sends the data encoded as a reduced representation. On UHDK5 the control points are sent directly and the phases are solved on the device.
