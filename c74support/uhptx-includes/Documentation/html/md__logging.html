Logging and Error Handling

The SDK includes built-in logging, which can be configured from within your application. This allows you to choose where messages are logged to, what level of messages you wish to log, and gives you the opportunity to handle specific error messages programmatically.

Considerations when using the logging system

The logging configuration is stored within an UltrahapticsLibrary object. Normally these objects are automatically created and destroyed by Emitters, but this can be problematic when you wish to maintain logging settings over the course of your program's lifetime.

We recommend that you create an UltrahapticsLibrary object before using the rest of the API, set up your logging settings, store the UltrahapticsLibrary object for the lifetime of your program, and destroy the object before exiting.

Log levels

The SDK has various log levels denoting the different levels of log message output:

Level Output
None (-1) No logging output under any circumstances.
Fatal Error (0) Only log fatal errors which will cause the SDK to immediately exit.
Error (1) Also log recoverable errors which still indicate a problem with the SDK's operation.
Warning (2) Also log warnings which might indicate minor problems with the SDK's operation or incorrect usage of the API.
Verbose (3) Also log informational messages which would occur under normal operation.
Extra Verbose (4) Also log very verbose messages which would occur under normal operation.

Please note that use of the Extra Verbose log level is very likely to impact performance, due to the frequency at which log entries are written.

Accessing and changing the logging configuration

The logging configuration is accessed by calling the getLoggingInterface() method of the UltrahapticsLibrary object.

From here you can change the log level:

Ultrahaptics::UltrahapticsLibrary uhLib;
uhLib.getLoggingInterface().setLogLevel(3);
uhLib.getLoggingInterface().redirectLoggingToStdOut();

Log messages can be redirected to the standard output or standard error streams, to a user-specified file, or discarded.

Using logging callbacks

For each log level, API users can register a callback to be called for every log message at that level. This allows users to take action based on messages received. To make this easier to use, many log messages have a specific error code attached to them, to indicate the specific situation that has occurred.

An example of creating a logging callback can be found below:

void log_callback(const char *tag, unsigned int message_level,
uint32_t error_code, const char *message, void *user_pointer)
{
if (error_code == (uint32_t)Ultrahaptics::ErrorCode::DEVICE_FAILED_TO_CLAIM)
{
// Handle case where device is in use by another process
}
}
void register_callback(Ultrahaptics::UltrahapticsLibrary* uhLib)
{
uhLib->getLoggingInterface().setCallbackForLogLevel(0, log_callback, nullptr);
uhLib->getLoggingInterface().setCallbackForLogLevel(1, log_callback, nullptr);
uhLib->getLoggingInterface().setCallbackForLogLevel(2, log_callback, nullptr);
}

This example would register the defined callback to run for any messages at the Fatal Error, Error or Warning level. The final iuser_pointer argument to setCallbackForLogLevel() allows the user to provide a "context" pointer, which is passed to the callback every time it is called. This pointer may be null if its use is not required.

Please note that the callback will be executed on the same thread used to process logging; this means that if the callback takes a long time to execute, further logging could be delayed. Any long-running actions required as a result of callback events should be taken on a separate thread.

Environment Variables

The SDK supports the following environment variables to control logging.

Variable Values
UH_LOG_LEVEL Integer log level. Lower priority than values set in the code unless UH_LOG_LEVEL_FORCE is set.
UH_LOG_DEST Where to log to. <filename>, stdout, stderr or null. Lower priority than values set in the code, unless UH_LOG_DEST_FORCE is set.
UH_LOG_LEVEL_FORCE Set to 1 to force the use of UH_LOG_LEVEL irrespective of what the code requests.
UH_LOG_DEST_FORCE Set to 1 to force the use of UH_LOG_DEST irrespective of what the code requests.
