Known Issues

  • Occasionally the Ultrahaptics Evaluation Kit (UHEV1) may fail to power up correctly. You can identify this as the status LED does not light up. If this happens, simply turn the power off and on again. This issue seems to be exacerbated by heat. If it happens multiple times in a row, leave the device to cool for a few minutes before trying again.

Release History

2.5.2 {v2_5_2}

  • Fixed possible crash/deadlock on active device disconnect
  • Fixed problem with finding Leap library when building examples on non-Windows platforms

2.5.1

  • Added support for Ultrahaptics STRATOS Explore (USX) devkit
  • Added mock device support for USX devkit
  • Improved performance of mock device file logging
  • Added several missing common methods to the .NET API's IEmitter interface
  • Added a new CMake variable ULTRAHAPTICS_RUNTIME_LIBRARY to allow finding of the Ultrahaptics runtime library (e.g. Ultrahaptics.dll) automatically
  • Windows: Fixed possible crash which could occur when device was disconnected while in use

2.5.0

  • Added support for Linux on x86 and x86_64 (initially supporting Ubuntu 16.04 LTS)
  • Added support for Linux on some ARM architectures: AArch64 (ARMv8) and ARMHF (ARMv7)
  • Added mock device support to allow testing the SDK without hardware connected
  • Added header and API call to retrieve SDK version information at runtime and/or compile-time
  • Added support for changing the update rate of emitters for advanced users
  • Fixed issue related to using multiple devices on a heavily-loaded USB bus
  • Updated examples to use new device info APIs
  • Windows: The SDK is now built with MSVC 14.1 (Visual Studio 2017)

2.4.0

  • Functionality has been added to the UltrahapticsLibrary and Emitter classes to allow querying of more detailed information about connected devices.
  • Moved to a new timing system which should significantly decrease CPU usage when using devices based on the UHEV1 platform.
  • Logging is now using spdlog, improving the format and performance of logging, especially on Windows.
  • Removed the Demo Suite application from the SDK installer. The Demo Suite is now available as a separate download on the Ultrahaptics Developer Site.
  • Updated UHFlashTool to version 1.1.0.
  • Updated documentation of examples.
  • Fixed an issue where the SDK could crash when starting an emitter after a UHEV1 was disconnected while in operation.
  • Fixed an issue related to Amplitude Modulation emitters on UHEV1, which caused louder than usual operation.
  • Windows: When using devices based on the UHEV1 platform the current process and emission callback thread priority will both be raised above default. This behaviour is configurable via methods on UltrahapticsLibrary.
  • Mac: When using devices based on the UHEV1 platform the emission callback thread priority will both be raised above default. This behaviour is configurable via methods on UltrahapticsLibrary.

2.3.2

  • UHDK5: Fixed a problem with the alignment configuration file for the rectangular board layout with the Leap Motion controller at the side

2.3.1

  • Mac: Fixed a problem causing the CMake package script to set an incorrect library path

2.3.0

  • UHDK5: Added support for rectangular board layout
  • Added support for defining bounding volumes outside of which the SDK will not produce points.
  • Added a Unity Example Project to the SDK Examples folder
  • UHEV1: Fixed an issue where when emitting more than two points, one or more points could appear to not be output, or be very weak.
  • Fixed an issue where connected devices might temporarily not be picked up by the SDK after rapidly destroying/recreating emitters or library objects.
  • Made AmplitudeModulation::Emitter copy and assignment constructors private. These were previously public but should not be used. Their use will now be detected at compile time. TimePointStreaming::Emitter's copy and assignment constructors were already correctly marked private.
  • Removed the AmplitudeModulation::ControlPoint::FrequencyContainer API. In theory this allowed multiple modulation frequencies for one control point but it has not been supported for some time. In the unlikely event that you were previously using ControlPoint::FrequencyContainer::add() please switch to ControlPoint::setFrequency() instead.

2.2.2

  • Fixed issue where emitter could deadlock on destruction if it was running and the callback was querying certain emitter parameters at the same time
  • Improve behaviour when trying to connect to separate devices at the same time on different threads
  • Fixed bug in the multiplication operator of the Matrix4x4 class which would cause it to produce incorrect results

2.2.1

  • Added two basic Unity examples
  • Fixed incorrect behaviour related to disabling control points on TimePoint Streaming emitters
  • Windows: Fixed an issue with the 32-bit SDK where Amplitude Modulation emitters could still become demodulated after many hours
  • Windows: Ship .lib files in the ZIP package as well as inside the installer, for easier access
  • Windows: Fix issue introduced in 2.2.0 where examples would not be installed correctly
  • Windows: Corrected the installer's title: it is no longer Evaluation Kit-specific

2.2.0

2.2.0-beta1

  • Updated the documentation, including a section on how mid-air haptics work
  • Windows: Fixed an issue where the ultrasound could become demodulated if using Amplitude Modulation emitters after many hours
  • Windows: We now recommend use of the libusbK driver on Windows 7, rather than WinUSB. This has been updated in our documentation, please see Installing USB driver on Windows 7
  • Windows: Fixed an issue where not all of the required C++ redistributables were installed with the 64-bit SDK installer

  • The Ultrahaptics Demo Suite has been reworked - now has much improved look and feel and integrates the new 3D Shapes demo

2.0.0-beta2

  • Added Windows 64-bit support (available as a separate SDK installer).
  • Lots of improvements to the source code examples provided with the SDK.
  • Added documentation on how to use audio files to implement custom modulation of control points (see Using custom modulation).
  • Added Ultrahaptics array firmware flashing instructions into the documentation (see Firmware Update Procedure).
  • The Ultrahaptics Demo which ships with the SDK has been improved. The fourth demo (dial demo) now has stopping & starting music playback with a tap gesture, and the volume is controlled with the dial motion before.

  • Various improvements and clarifications in the API documentation.

2.0.0-beta1

  • Introduced new V2 API with new TimePointStreaming emitter.
  • The way decibel safety limiting is applied to control point intensity has been improved. Previously, setting intensities above ~0.6 would have no effect upon the output intensity. Now, intensity is easier to work with, as an intensity of 1.0 always represents the highest safe output power.
  • The default (evaluation kit) Ultrahaptics array alignment data is now stored internally in the SDK, fixing instability when loading the default alignment from the disk on some platforms. Custom alignment files can still be loaded.
  • The code examples have been rewritten, thoroughly documented, and extended with additional examples. The 'Ultrahaptics Editor' application has been removed.
  • Fixed documentation referring to the Ultrahaptics array axes incorrectly.
  • Improved instructions for 'dial demo' in Ultrahaptics demo application.
