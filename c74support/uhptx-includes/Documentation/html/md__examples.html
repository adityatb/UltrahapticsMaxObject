We provide example programs to help get you up and running with the Ultrahaptics API. They are divided into those using Amplitude Modulation and Time Point Streaming*. These are summarised below and listed in order of increased complexity. You can find the source code in the Examples folder of the SDK download file.

AmplitudeModulation Examples

  • AmplitudeModulation_Focus Projects a focal point, fixed 20cm above the centre of the array.
  • AmplitudeModulation_HandTracking Track the hand and project a point onto the palm.
  • AmplitudeModulation_Dial Projects a rotating "dial" onto the palm and uses a gesture to control its value.

TimePointStreaming Examples

  • TimePointStreaming_Focus Projects a static control point, fixed above the centre of the array (point data generated on-the-fly).
  • TimePointStreaming_CachedPoint Projects a static control point above the centre of the array (point data precomputed).
  • TimePointStreaming_Circle Projects a 4cm circle, fixed above the centre of the array.
  • TimePointStreaming_Forcefield Generate a "forcefield" effect on the hand centred across the array.
  • TimePointStreaming_Freeform Projects a static point fixed above the centre of the array, using a custom waveform read from an input file (e.g. the provided Sample_Wave.wav).
  • TimePointStreaming_FreeformTracking Adds tracking to the TimePointStreaming_Freeform example.
  • TimePointStreaming_MultiPoint Uses two control points at the same time to project two diagonal lines onto the palm.

Building the Examples

In addition to the Ultrahaptics SDK, you will need to install the Leap Motion SDK for hand tracking examples, and software build tools.

CMake

We provide CMakeLists.txt for building with CMake, but you can use another build system if you wish. CMake is an extensible, open-source system that manages the build process and is used to create the platform make and project files needed to compile and run the examples. It can be installed directly by downloading from cmake.org/download or using Homebrew on Mac OS, or Chocolatey on Windows. Once installed, cmake can be run from the command line or via the GUI tool cmake-gui (preinstalled with Windows).

Leap Motion Controller SDK

The examples use the Leap Motion Controller, so download the Leap Motion V2 SDK. You will have to register and accept Leap Motion’s terms and conditions first. Leap Motion also provides a newer version of their SDK called Orion, available to Windows users (we recommend Orion for VR/AR applications). Unpack the downloaded package, including the LeapSDK folder, to your workspace and run the installer. See below for how to configure your build with the Leap Motion SDK. You can read the included Readme.txt file for more information.

MacOS

To install CMake with Homebrew, type: 

brew install cmake

Verify the installation by typing: 

cmake --version

Download and install XCode from the App Store or, to use the IDE of your choice, get the XCode Command Line Tools by typing xcode-select --install from a terminal and following the instructions (requires minimum OS X 10.9).

Create a build directory where you want the example code to be built. This is where we will run all cmake commands. Configure your build as follows (replace path/to with your Examples directory location): 

cd path/to/Examples
mkdir build
cd build
cmake ..

Note, cmake complains that it cannot find 'Leap'. Either run ccmake to set the path to the Leap include folder and library file manually, or copy each to a folder that will always be searched:

  • To configure with ccmake, type: 

      ccmake .

    Type 't' to toggle to advanced mode and scroll until you see the entries for LEAP_INCLUDE_DIR and LEAP_LIBRARY. You will find the include directory and libLeap.dylib library located inside your LeapSDK directory. Press enter to edit and set, with full path, for each. Type 'c' to configure, 'e' to exit advanced mode, and 'g' to generate and exit.

  • Alternatively, copy the header (.h) files from the include directory to /usr/local/include and libLeap.dylib to /usr/local/lib, and run cmake again. The advantage to this approach is that it only needs to be done once for any project using the Leap Motion Controller on the same machine.

You will now have a Makefile that can be used to build the examples: type make at the prompt (you can speed things up by using make -j4 to take advantage of your machine’s parallel processors). If everything has worked your build directory will contain executable files for each of the examples listed above. Connect your Ultrahaptics hardware and run each of these from the command line. For example: 

./AmplitudeModulation_Focus

Note that TimepointStreaming_Freeform and TimepointStreaming_FreeformTracking take an argument for a data file name: 

./TimepointStreaming_Freeform ../Sample_Wave.wav

Windows

To build on Windows, install either Visual Studio, or the Visual C++ Build Tools. These come with their own command prompts, correctly configured with the appropriate environment, and you should use these for any command line instructions. Shortcuts to these should be in your Start Menu, or can be found at C:\Program Files (x86)\Microsoft Visual C++ Build Tools.

You can use cmake from the command line, or the cmake-gui to generate project files for each example. We recommend using the cmake-gui which, when first started, will prompt you for the source path and your build directory (cmake-gui will create one if you haven’t done so yourself), then click configure. You will be prompted to select the build tools for your project from a drop-down box the first time you configure it.

Important: Select the correct build tools for your platform. You can find this by typing cl in the command prompt: the name and usage instructions for the compiler will be printed. Note that you must use the same compiler bit-architecture as the Ultrahaptics SDK you have installed, e.g Visual Studio 2015 Win64 if you have the 64-bit SDK installed. Most build errors relate to selecting the wrong version of compiler in the tool.

Note that the 'Leap' will not be found. Define the LEAP_ROOT path variable to point to the LeapSDK directory. This can be added as an entry using cmake-gui (click Add Entry, then configure again and generate again) or as a command-line argument with cmake such that: 

LEAP_ROOT: path\to\LeapSDK

Where path\to is your directory path. If using cmake from the command prompt, configure and generate by using the following: 

cd path\to\Examples
mkdir build
cd build
cmake -G"Visual Studio <your compiler>" -DLEAP_ROOT="path\to\LeapSDK" ..

This will generate the necessary project files for Visual Studio. You can open these to build and run as with any other Visual Studio project (ensure you have set the correct architecture for your platform from the drop-down box). 

Visual C++ Build Tools

If you have only installed Visual C++ Build Tools, cmake can be used to generate NMake files and by typing: 

cmake -G"NMake Makefiles" -DLEAP_ROOT="path\to\LeapSDK" ..

after which, you can build from the command line by typing: 

cmake --build . --config Release

Note that when building with Visual C++ Build Tools:

  • You can only build a release version of the code (a debugger can be installed separately).
  • cmake-gui should be launched from the command prompt installed with the tools to pickup the correct tools paths.

Running the examples

To run the example applications, you must copy the appropriate Leap.dll, Ultrahaptics.dll and libusb-1.0.dll files to the build directory. You can find Ultrahaptics.dll and libusb-1.0.dll in your Ultrahaptics installation under the bin folder. The Leap.dll can be found in the appropriate LeapSDK/lib subfolder (either x64 or x86).

Most build problems can be rectified by ensuring your build is in a separate directory, which can then be deleted if something goes wrong. Alternatively you can delete the cache in cmake-gui from the menu to reset and start again.

*Time Point Streaming is not supported by the TOUCH Development Kit (UHDK5).
